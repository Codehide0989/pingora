## CLI interface - pingora

pingora is a command line interface for managing your monitors and triggering your synthetics tests.

This is pingora Command Line Interface, the pingora.dev CLI.

Usage:

```bash
$ pingora [COMMAND] [COMMAND FLAGS] [ARGUMENTS...]
```

### `monitors` command

Manage your monitors.

Usage:

```bash
$ pingora [GLOBAL FLAGS] monitors [ARGUMENTS...]
```

### `monitors apply` subcommand

Create or update monitors.

> pingora monitors apply [options]

Creates or updates monitors according to the pingora configuration file.

Usage:

```bash
$ pingora [GLOBAL FLAGS] monitors apply [COMMAND FLAGS] [ARGUMENTS...]
```

The following flags are supported:

| Name                        | Description                                           |   Default value   |  Environment variables |
|-----------------------------|-------------------------------------------------------|:-----------------:|:----------------------:|
| `--config="…"` (`-c`)       | The configuration file containing monitor information | `pingora.yaml` |         *none*         |
| `--access-token="…"` (`-t`) | pingora API Access Token                           |                   | `PINGORA_API_TOKEN` |
| `--auto-accept` (`-y`)      | Automatically accept the prompt                       |      `false`      |         *none*         |


### `monitors import` subcommand

Import all your monitors.

> pingora monitors import [options]

Import all your monitors from your workspace to a YAML file; it will also create a lock file to manage your monitors with 'apply'.

Usage:

```bash
$ pingora [GLOBAL FLAGS] monitors import [COMMAND FLAGS] [ARGUMENTS...]
```

The following flags are supported:

| Name                        | Description                 |   Default value   |  Environment variables |
|-----------------------------|-----------------------------|:-----------------:|:----------------------:|
| `--access-token="…"` (`-t`) | pingora API Access Token |                   | `PINGORA_API_TOKEN` |
| `--output="…"` (`-o`)       | The output file name        | `pingora.yaml` |         *none*         |

### `monitors info` subcommand

Get a monitor information.

> pingora monitors info [MonitorID]

Fetch the monitor information. The monitor information includes details such as name, description, endpoint, method, frequency, locations, active status, public status, timeout, degraded after, and body. The body is truncated to 40 characters.

Usage:

```bash
$ pingora [GLOBAL FLAGS] monitors info [COMMAND FLAGS] [ARGUMENTS...]
```

The following flags are supported:

| Name                        | Description                 | Default value |  Environment variables |
|-----------------------------|-----------------------------|:-------------:|:----------------------:|
| `--access-token="…"` (`-t`) | pingora API Access Token |               | `PINGORA_API_TOKEN` |

### `monitors list` subcommand

List all monitors.

> pingora monitors list [options]

List all monitors. The list shows all your monitors attached to your workspace. It displays the ID, name, and URL of each monitor.

Usage:

```bash
$ pingora [GLOBAL FLAGS] monitors list [COMMAND FLAGS] [ARGUMENTS...]
```

The following flags are supported:

| Name                        | Description                               | Default value |  Environment variables |
|-----------------------------|-------------------------------------------|:-------------:|:----------------------:|
| `--all`                     | List all monitors including inactive ones |    `false`    |         *none*         |
| `--access-token="…"` (`-t`) | pingora API Access Token               |               | `PINGORA_API_TOKEN` |

### `monitors trigger` subcommand

Trigger a monitor execution.

> pingora monitors trigger [MonitorId] [options]

Trigger a monitor execution on demand. This command allows you to launch your tests on demand.

Usage:

```bash
$ pingora [GLOBAL FLAGS] monitors trigger [COMMAND FLAGS] [ARGUMENTS...]
```

The following flags are supported:

| Name                        | Description                 | Default value |  Environment variables |
|-----------------------------|-----------------------------|:-------------:|:----------------------:|
| `--access-token="…"` (`-t`) | pingora API Access Token |               | `PINGORA_API_TOKEN` |

### `run` command (aliases: `r`)

Run your synthetics tests.

> pingora run [options]

Run the synthetic tests defined in the config.pingora.yaml.

Usage:

```bash
$ pingora [GLOBAL FLAGS] run [COMMAND FLAGS] [ARGUMENTS...]
```

The following flags are supported:

| Name                        | Description                 |      Default value       |  Environment variables |
|-----------------------------|-----------------------------|:------------------------:|:----------------------:|
| `--config="…"`              | The configuration file      | `config.pingora.yaml` |         *none*         |
| `--access-token="…"` (`-t`) | pingora API Access Token |                          | `PINGORA_API_TOKEN` |

### `whoami` command (aliases: `w`)

Get your workspace information.

> pingora whoami [options]

Get your current workspace information, display the workspace name, slug, and plan.

Usage:

```bash
$ pingora [GLOBAL FLAGS] whoami [COMMAND FLAGS] [ARGUMENTS...]
```

The following flags are supported:

| Name                        | Description                 | Default value |  Environment variables |
|-----------------------------|-----------------------------|:-------------:|:----------------------:|
| `--access-token="…"` (`-t`) | pingora API Access Token |               | `PINGORA_API_TOKEN` |
